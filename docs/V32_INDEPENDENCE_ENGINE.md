# NimbrayAI V32 — Independence Engine

La V32 rend NimbrayAI plus autonome : le cerveau local répond d'abord aux demandes stables, fréquentes, humaines ou sensibles, et Groq n'est appelé que pour les demandes réellement complexes.

## Objectifs

- Réduire les appels inutiles à Groq.
- Rendre les micro-dialogues instantanés et humains.
- Ajouter plus de variations pour éviter l'effet robot.
- Renforcer les connaissances locales stables.
- Masquer les erreurs techniques et éviter les JSON d'API visibles.
- Garder Groq pour la création, l'analyse, le code et les demandes personnalisées.

## Ce qui reste local

- ok, merci, bof, haha, pas mal, stop, parle pas, ta gueule ;
- sécurité humaine : suicide, violence, meurtre, illégal ;
- science simple : ciel bleu, mer salée, électricité, atomes, pluie ;
- cuisine simple ;
- obsèques, décès, héritage, plainte ;
- sport, langues, cultures, motivation, confiance, cybersécurité défensive ;
- identité et limites de NimbrayAI.

## Ce qui appelle Groq

- demandes longues ;
- rédaction personnalisée ;
- analyse complexe ;
- code non trivial ;
- stratégie ;
- création originale ;
- raisonnement avec contexte.

## Philosophie

NimbrayAI doit donner l'impression d'une discussion avec une personne attentive : simple, vive, rassurante, parfois drôle, mais toujours sérieuse quand le sujet l'exige.
